# Useful Linux Resources

## General Interest

- [How To Ask Questions the Smart Way](https://www.catb.org/~esr/faqs/smart-questions.html) is a timeless guide on making it easier and more pleasant for open source community members to help you work through problems you encounter.  Yes, the SSL cert is broken again.
- [CLI Magic](https://www.climagic.org) is a social media feed ([X](http://www.twitter.com/climagic) and [Mastodon](http://mastodon.social/@climagic)) featuring a new command-line tip or trick each day.  The web site has some fun extras.  It’s all maintained by a Bloominglabs founding member and Bloomington local, Mark Krenz.
- [Cheatography](https://cheatography.com) offers free printable refcards (reference cards) for many open source tools, programming languages, and more.  You can also create your own refcards using their handy layout tool.
- [Linux101.dev](https://linux101.dev) offers an online command reference that is made of very simple HTML so you can easily use it from the command line if you don’t have a GUI browser handy.
- [Alternative.to](https://get.alternative.to) is a website that answers the question “I used to use X, what else could I do that job with?”.  Search for the application you were familiar with before, click on its listing, and see what other users have tried replacing it with, complete with reviews.  This isn’t limited to Linux software, but it’s a handy way for new Linux users to find Linux-native alternatives to tools they used to use on a different OS.
- [Humble Bundle](https://www.humblebundle.com/books) offers a DRM-free bundle of Linux-related ebooks a couple of times per year.  If you want an affordable way to build your personal tech library, this is great.  I especially like the older O’Reilly reference manuals and anything from No Starch Press.
- [DistroWatch](https://distrowatch.com) is a decent place to start poking around when you get sick of your Linux Distro.  DistroWatch aims to catalog as many Linux and open source UNIX or POSIX OSes as possible.

## Distro-Specific

Your distribution’s web site, be that [Ubuntu](https://ubuntu.com), [Linux Mint](https://linuxmint.com), [Fedora](https://fedoraproject.org), [Gentoo](https://gentoo.org), [NixOS](https://nixos.org), [Arch](https://archlinux.org), or something else, is a great jumping off point for documentation and support.

- Look for links called **Documentation** or **Wiki** or **Manual** to find the core documentation for your operating system.
- Look for links called **Community** or **Forum** or **Discourse** to find the community forum for news, discussion, and Q&A.
- Look for links called **Community Chat** or **Matrix** or **IRC** or **Discord** to find the community chat room(s).
- Poke around to find so much more.  Every distro is a bit different, but most have some sort of community, user documentation, developer documentation, code repository, and so on.
